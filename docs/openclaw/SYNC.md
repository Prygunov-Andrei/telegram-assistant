# Синхронизация задач Mac <-> VPS

Двухсторонняя синхронизация задач между Obsidian на Mac и ботом на VPS.

## Обзор

```
Mac (~/obsidian/)          VPS (/root/obsidian-tasks/)
├── april/ЗАДАЧИ/    ←→    ├── april/ЗАДАЧИ/
├── avgust/ЗАДАЧИ/   ←→    ├── avgust/ЗАДАЧИ/
├── books/ЗАДАЧИ/    ←→    ├── books/ЗАДАЧИ/
├── ...              ←→    ├── ...
└── ДАШБОРД.md       ←→    └── ДАШБОРД.md
```

**Принцип:** `rsync --update` — побеждает файл с более новой датой модификации. Конфликтов нет (один пользователь).

**Ограничение:** удаление файлов НЕ синхронизируется автоматически. Удалять нужно вручную на обеих сторонах.

## Скрипты

### sync-tasks.sh (основной)

Полный цикл: pull + push. Используется для регулярной синхронизации.

```bash
bash scripts/sync-tasks.sh
```

Сначала тянет изменения с VPS (pull), потом отправляет свои (push). `rsync --update` на обоих шагах.

### pull-tasks.sh

Только VPS → Mac. Используется когда нужно забрать что наделал бот.

```bash
bash scripts/pull-tasks.sh
```

**Внимание:** использует `--delete` — файлы, удалённые на VPS, будут удалены и на Mac.

### push-tasks.sh

Только Mac → VPS. Используется когда обновил задачи в Obsidian.

```bash
bash scripts/push-tasks.sh
```

**Внимание:** использует `--delete` — файлы, удалённые на Mac, будут удалены и на VPS.

## Автоматизация

### Текущее состояние

Автоматической синхронизации (cron) **нет**. Скрипты запускаются вручную.

Бот знает о синхронизации (MEMORY.md и TOOLS.md пишут "каждые 5 минут через cron"), но фактически crontab на обеих сторонах пуст.

### Как добавить автосинхронизацию

На Mac (crontab):
```bash
crontab -e
# Каждые 5 минут:
*/5 * * * * /bin/bash /Users/andrei_prygunov/obsidian/telegram-assistant/scripts/sync-tasks.sh >> /tmp/sync-tasks.log 2>&1
```

На VPS cron не нужен — VPS всегда имеет актуальную версию (бот пишет напрямую).

## Известные проблемы

1. **pull-tasks.sh и push-tasks.sh используют `--delete`** — могут удалить файлы на целевой стороне
2. **sync-tasks.sh использует `--update` (безопаснее)** — не удаляет, только обновляет более новые
3. **MEMORY.md/TOOLS.md бота утверждают что sync работает по cron** — это не так, нужно либо настроить cron, либо обновить документы бота
