# Workspace Memory

OpenClaw использует файловую систему в workspace как "память" агента. Бот читает и пишет файлы для хранения контекста между сессиями.

## Как работает память

### Загрузка при старте сессии

При каждой новой сессии (hook `boot-md`) агент получает содержимое:
1. `MEMORY.md` — долгосрочная память (правила, факты)
2. `TOOLS.md` — справка по инструментам

### Сохранение контекста (hook `session-memory`)

В конце сессии агент может обновить файлы в workspace, сохраняя новую информацию.

## Файлы памяти

### MEMORY.md (долгосрочная)

Содержит ключевые правила для агента:
- Система задач: путь, синхронизация, правила
- "Создавать задачи сразу в `/root/obsidian-tasks/`"
- "rsync --update — побеждает более новый файл"
- "Удаление НЕ синхронизируется"
- "Если Андрей говорит 'обновил задачи' — перечитать файлы"

### TOOLS.md (справка)

Справочные данные:
- Пути к задачам, дашборду, recurring
- SSH адрес VPS
- TTS настройки (ElevenLabs Ivan)
- Timezone (Europe/Berlin)
- Ссылка на `skills/tasks/SKILL.md`

### memory/ (дневные записи)

| Файл | Содержимое |
|------|------------|
| `YYYY-MM-DD.md` | Утренний лог (подъём, вес), питание (КБЖУ по приёмам) |
| `shopping-YYYY-MM-DD.md` | Покупки с ценами, магазинами и КБЖУ |
| `shopping-registry.md` | Реестр часто покупаемых продуктов с ценами |
| `dreams.md` | Записи снов с анализом (интерпретация символов) |

### nutrition/ (профиль питания)

| Файл | Содержимое |
|------|------------|
| `PROFILE.md` | Антропометрия, цели, суточный бюджет КБЖУ |
| `log/YYYY-MM-DD.md` | Дневник питания за день |

## Как кастомизировать

### Добавить новую память

Создать файл в `/root/.openclaw/workspace/memory/` или добавить информацию в `MEMORY.md`.

### Добавить инструкцию для агента

Добавить правило в `MEMORY.md` или `TOOLS.md`. Агент прочтёт их при старте следующей сессии.

### Добавить новый tracking

1. Создать папку в workspace (например `workspace/fitness/`)
2. Описать формат в `TOOLS.md` или в отдельном skill
3. Агент начнёт использовать формат при обращении

## Git-история workspace

Workspace — git-репо. История изменений:

```bash
cd /root/.openclaw/workspace
git log --oneline
# 36b0c70 Update nutrition profile
# be1cdf4 Add nutrition tracking
# 3f9bc56 Remove BOOTSTRAP.md
# 9d30e54 Bootstrap: identity and user profile
```

Можно откатить изменения при необходимости.

## Важно

- Агент имеет полный read/write доступ к workspace (tool profile: `coding`)
- Агент может создавать новые файлы и директории
- `MEMORY.md` и `TOOLS.md` загружаются автоматически — остальные файлы агент читает по запросу
- Файлы в `memory/` — это не автоматический лог, а записи, которые агент создаёт осознанно по инструкциям
