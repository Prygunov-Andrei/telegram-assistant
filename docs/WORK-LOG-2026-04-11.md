## 3. Voice Sidecar v2 — мгновенная транскрибация

### Проблема
OpenClaw pipeline для голосовых занимал 36+ секунд. Groq Whisper сам по себе — 0.3 сек.

### Решение
Отдельный Python-сервис, минующий OpenClaw pipeline.

**Файл:** `vps/voice-sidecar/main.py`

**Как работает:**
1. `inotifywait` следит за `/root/.openclaw/media/inbound/`
2. Новый `.ogg` → сразу Groq Whisper (не ждёт OpenClaw)
3. `.txt` транскрипт → отправка в Telegram через Bot API
4. Удаление оригинального голосового через `deleteMessage`

**Результат:** 1-2 сек от отправки до текста (вместо 36+).

**Systemd:** `vps/systemd/voice-sidecar.service`

## 4. Синхронизация задач через GitHub

### До
rsync через SSH между Mac и VPS, запуск вручную.

### После
Двусторонняя синхронизация через GitHub:

```
VPS bot → git push → GitHub → git pull (cron Mac) → rsync → Obsidian
                            ↑
                        Mac push → git push → GitHub → git pull (VPS)
```

**Компоненты:**
- GitHub репо: `Prygunov-Andrei/obsidian-tasks` (приватный)
- Mac cron: каждые 2 минуты
- Скрипт: `scripts/sync-tasks-git.sh`
- Env: `~/.github-tasks-sync.env` (секреты, не в git)
- Бот команды: `/sync` в Telegram, кнопка PUSH/PULL в Mini App

**Гарантии без потери данных:**
- `git pull --ff-only` — падает при конфликте, не перезаписывает
- `rsync -a --update` — нет `--delete`, файлы не исчезают
- GitHub = облачный бэкап

## 5. Telegram Mini App

### Архитектура
```
iPad Telegram → WebApp iframe
    ↕ HTTPS (hvac-news.online)
Caddy → FastAPI (:8443)
    ↕ читает/пишет .md файлы
/root/obsidian-tasks/ (git repo)
    ↕ git push/pull
GitHub → Mac Obsidian
```

### Backend — `vps/task-webapp/`
**4 Python файла (~500 строк):**

| Файл | Назначение |
|------|------------|
| `main.py` | FastAPI endpoints: `/api/tasks`, `/api/task/{project}/{id}`, `/api/transcribe`, `/api/sync` |
| `vault.py` | YAML frontmatter парсинг, CRUD задач, `regenerate_dashboard()` |
| `auth.py` | Telegram initData HMAC-SHA256 валидация |
| `git_sync.py` | `pull()`, `commit_and_push()`, `ensure_remote()` |

### Frontend — `vps/task-webapp/static/index.html`
Single-page app, vanilla JS + Telegram WebApp SDK.

**UI:**
- Принудительно тёмная тема (игнорирует Telegram `themeParams`)
- Sidebar (250px): "Сегодня!" сверху, затем "Все проекты", проекты с счётчиками
- Второй sidebar (250px): теги/подразделы выбранного проекта
- Таблица задач: ID, Название, Статус, Дедлайн, Проект, Исполнитель
- Фильтр по статусу
- Safe-area insets (viewport-fit=cover) — нет светлых полосок в fullscreen
- Модалки: создание и редактирование
  - Textarea для markdown body
  - Tag picker с чипами (добавить из существующих или новый)
- Swipe на задачах: влево = архив, вправо = cancel
- Кеш задач на 30 сек → фильтрация локально

### Слайдеры вместо кнопок
**Главный навбар** (внизу экрана):
```
ПУЛ   ПУШ   Создать   [ЗАПИСЬ!]
             (слайдер движется между 4 позициями)
```
- Перетаскивание или тап на подпись
- Тап на кнопку = выполняет действие текущей позиции
- По умолчанию: ЗАПИСЬ! (справа)

**Модалки задач** (3-позиционный):
```
Отмена  ←  Сохранить/Создать  →  Запись!
         (слайдер по центру по умолчанию)
```
- Запись! → голос → транскрипт вставляется в textarea на позиции курсора
- Повторная запись = дописывает в месте курсора (как macOS dictation)

### Инфраструктура
- **Домен:** `hvac-news.online` (напрямую на VPS, без Cloudflare)
- **Caddy** reverse proxy с Let's Encrypt SSL
- **systemd:** `task-webapp.service`
- **No-cache headers** на HTML (Caddy)
- **BotFather:** Menu Button "Задачи" → Mini App URL

### Endpoints
| Method | Path | Что делает |
|--------|------|------------|
| GET | `/api/projects` | Список проектов + счётчики |
| GET | `/api/tasks` | Все задачи (фильтры: project, status) |
| GET | `/api/task/{project}/{id}` | Задача с body |
| POST | `/api/task` | Создать |
| PATCH | `/api/task/{project}/{id}` | Обновить статус/дату/теги/body |
| DELETE | `/api/task/{project}/{id}` | Архивировать |
| POST | `/api/transcribe` | Audio → текст через Groq |
| POST | `/api/sync` | git pull/push |
